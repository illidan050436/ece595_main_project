Spanning Tree Algorithm Overview


1. Motivation:

Spanning tree alogrithm is a solution to the loop problems in network address learning. It can maintain a spanning tree for given topology in a dynamic environment. The following part would mainly focus on the distributed algorithms.

2. Terminologies:

-Root Bridges: The bridge with the lowest identifier is the root of the spanning tree.

-Root Path Cost: For each bridge, the cost of the min-cost path to the root, measured in number of hops to the root.

-BPDU: Configuration bridge protocol data unit, a message being sent between bridges that contains root ID, root path cost and bridge ID information.

3. Principles:

-Initially all bridges assume theay are the root bridge. Each bridge sends their BPDUs on all its ports.

-Upon receiving BPDU on a port, bridge checks if received one can lead to a better mapping. If any BPDU received is considered better, then the bridge will converge, namely taking the root ID in BPDU as its own root bridge, adding 1 to the cost.

-When comparing two BPDUs M1 and M2 with the form (R1|C1|B1) and (R2|C2|B2), the following ordering principles would apply:

If (R1 < R2)

M1 better than M2

elseif ((R1 == R2) & (C1 < C2))

M1 better than M2

elseif ((R1 == R2) & (C1 == C2) & (B1 < B2))

M1 better than M2

else

M2 better than M1

-After a bridge has done the message processing and generated its new BPDU, it will only forward this BPDU on one of its ports only if any BPDUs received from that port is worse than its new BPDU.
