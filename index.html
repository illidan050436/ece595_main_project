Spanning Tree Algorithm Overview

  • Motivation

    • Spanning tree alogrithm is a solution to the loop problems in network address learning. It can maintain a spanning tree for given topology in a dynamic environment. The following part would mainly focus on the distributed algorithms.

  • Terminologies
  • -Root Bridges:
    The bridge with the lowest identifier is the root of the spanning tree.
  • -Root Port:
    Each bridge has a root port which identifies the next hop from a bridge to the root.
  • -Root Path Cost:
    For each bridge, the cost of the min-cost path to the root, measured in number of hops to the root.
  • Terminologies
  • -Designated Bridge:
    Bridge on LAN that provides minimal cost path to root for this LAN. If two bridges have the same cost, select one with lower identifier.
  • -Designated Port:
    If a bridge is the designated bridge for a LAN, the appropriate port is the designated port.
  • -BPDU:
    Configuration bridge protocol data unit, a message being sent between bridges that contains root ID, root path cost and bridge ID information.
  • Principle
  • -Initially all bridges assume theay are the root bridge. Each bridge sends their BPDUs on all its ports.
  • -Upon receiving BPDU on a port, bridge checks if received one can lead to a better mapping.
  • -If any BPDU received is considered better, then the bridge will converge, namely taking the root ID in BPDU as its own root bridge, adding 1 to the cost.

  • Ordering of Messages
  • -When comparing two BPDUs M1 and M2, the ordering principles on the right would apply.
  • -M1 and M2 are with the form [R1 | C1 | B1] and [R2 | C2 | B2], in which correspond to (root bridge id|root path cost|sending bridge id)
  • -After a bridge has done the message processing and generated its new BPDU, it will only forward this BPDU on one of its ports only if any BPDUs received from that port is worse than its new BPDU.
  • Ordering Principles
  • -We can order BPDU messages with the following ordering relation, icon means better than.
  • If (R1 < R2)
  • M1 M2
  • elseif ((R1 == R2) & (C1 < C2))
  • M1 M2
  • elseif ((R1 == R2) & (C1 == C2) & (B1 < B2))
  • M1 M2
  • else
  • M2 M1
